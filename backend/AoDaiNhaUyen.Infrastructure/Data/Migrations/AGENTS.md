## For AI Agents
### Working In This Directory
- Do NOT manually edit migration files -- use `dotnet ef migrations add <Name>` to generate new ones
- Run from Infrastructure project: `dotnet ef migrations add <Name> --startup-project ../AoDaiNhaUyen.Api`
- Migrations are applied automatically on startup when `RunMigrationsAndSeedOnStartup` is true
- The snapshot file (`AppDbContextModelSnapshot.cs`) reflects the current state of the model -- it is regenerated with each new migration
- To rollback: `dotnet ef database update <TargetMigration>` or remove the last migration with `dotnet ef migrations remove`
- Connection string is resolved by `AppDbContextFactory` from Api project's appsettings or env vars
